# Руководство по внесению вклада

Спасибо за интерес к проекту! Мы рады любым вкладам в развитие `@makebelieve21213-packages/redis-client`.

## Как внести вклад

### Сообщение об ошибках

Если вы нашли ошибку:

1. Проверьте, не была ли она уже зарегистрирована в [Issues](https://github.com/makebelieve21213-packages/redis-client/issues)
2. Если нет, создайте новую issue с описанием:
   - Что произошло
   - Ожидаемое поведение
   - Шаги для воспроизведения
   - Версия Node.js, pnpm
   - Версия пакета

### Предложение улучшений

Если у вас есть идея по улучшению:

1. Создайте issue с описанием предложения
2. Объясните, почему это улучшение будет полезным
3. Опишите возможную реализацию (если есть идеи)

### Pull Requests

1. **Форкните репозиторий** и создайте ветку для ваших изменений:
   ```bash
   git checkout -b feature/amazing-feature
   # или
   git checkout -b fix/bug-description
   ```

2. **Убедитесь, что код соответствует стандартам**:
   ```bash
   # Установите зависимости (Husky настроится автоматически)
   pnpm install
   
   # Запустите линтер
   pnpm lint
   
   # Запустите форматирование
   pnpm format:fix
   
   # Запустите тесты
   pnpm test
   ```

   **Примечание**: Проект использует Husky для автоматической проверки кода перед коммитом и push. При создании коммита автоматически запускаются проверки форматирования и линтера. При push запускаются тесты. Это помогает поддерживать качество кода.

3. **Добавьте тесты** для новых функций или исправлений багов

4. **Убедитесь, что все тесты проходят**:
   ```bash
   pnpm test:coverage
   ```

5. **Обновите документацию** (README.md, llms.txt) если необходимо

6. **Создайте Pull Request** с описанием изменений

## Стандарты кода

### TypeScript

- Используйте TypeScript для всего кода
- Избегайте использования типа `any`
- Добавляйте типы для всех функций и переменных
- Следуйте существующему стилю кода

### Форматирование

Проект использует Prettier для форматирования. Перед коммитом запустите:

```bash
pnpm format:fix
```

### Линтинг

Проект использует ESLint. Убедитесь, что код проходит проверку:

```bash
pnpm lint
```

### Тесты

- Все новые функции должны иметь тесты
- Стремитесь к 100% покрытию кода
- Используйте Jest для тестирования
- Тесты должны быть понятными и независимыми

### Коммиты

Используйте понятные сообщения коммитов:

```
feat: добавить поддержку кастомных стратегий кэширования
fix: исправить ошибку с connection pooling
docs: обновить README с примерами использования
test: добавить тесты для нового функционала
refactor: упростить логику обработки соединений
```

## Процесс ревью

1. После создания PR, изменения будут проверены
2. Могут быть запрошены изменения
3. После одобрения PR будет влит в основную ветку

## Вопросы?

Если у вас есть вопросы, создайте issue или свяжитесь с maintainer'ами проекта.

Спасибо за ваш вклад! 🎉
